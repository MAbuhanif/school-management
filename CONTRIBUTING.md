# Contributing to School Management System

Thank you for considering contributing to the School Management System!

## Code of Conduct

This project adheres to a Code of Conduct. By participating, you are expected to uphold this code. Please report unacceptable behavior to the project maintainers.

## How to Contribute

1.  **Fork the Repository**: Create a fork of the project to your own GitHub account.
2.  **Clone the Fork**: Clone your fork to your local machine.
3.  **Create a Branch**: Create a new branch for your feature or bug fix.
    ```bash
    git checkout -b feature/amazing-feature
    ```
4.  **Make Changes**: Implement your changes. Ensure your code follows the project's coding standards.
5.  **Run Tests**: Run existing tests and add new ones if applicable.
    ```bash
    php artisan test
    ```
6.  **Commit Changes**: Commit your changes with clear, descriptive messages.
    ```bash
    git commit -m "feat: add amazing feature"
    ```
7.  **Push to Fork**: Push your branch to your fork.
    ```bash
    git push origin feature/amazing-feature
    ```
8.  **Submit a Pull Request**: Open a Pull Request from your fork to the main repository.

## Coding Standards

- **PHP**: Follow PSR-12 coding standards.
- **JavaScript/React**: Follow standard React best practices and use functional components with Hooks.
- **CSS**: Use Tailwind CSS utility classes.

## Reporting Bugs

If you find a bug, please open an issue using the Bug Report template. Include as much detail as possible to help us reproduce and fix the issue.
